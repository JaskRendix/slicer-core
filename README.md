# Mesh Slicing Engine  
A compact slicing core for generating 2D contours and toolpath primitives from 3D triangle meshes.

This project implements the geometry stages found in 3D‑printing slicers.  
It focuses on correctness, modularity, and clarity.  
The codebase is small and easy to extend.

---

## Features

### Mesh and Geometry
- Triangle–plane intersection  
- Multi‑layer slicing  
- Segment stitching into continuous polylines  
- Winding detection (CW/CCW)  
- Winding normalization (force outer CCW, holes CW)  
- Island construction (outer contour with assigned holes)

### Offsetting and Perimeters
- Robust polyline offsetting  
- Inward and outward offsets  
- Perimeter generation with configurable shell count

### Infill
- Line infill  
- Grid infill  
- Hex infill  
- Clipping against outer contours and holes

### Reliability
- Degenerate geometry handling  
- Deterministic behavior  
- Unit tests for slicing, reconstruction, and edge cases

## Core Concepts

### Triangle–Plane Intersection
Each triangle is intersected with a horizontal plane.  
Edges that cross the plane produce intersection points through linear interpolation.

### Layer Slicing
All triangles are processed at each Z height.  
Valid intersections form unordered line segments.

### Polyline Reconstruction
Segments are stitched by endpoint matching.  
This produces closed loops or open polylines.

### Winding and Islands
Contours are classified by orientation.  
Outer loops are normalized to CCW.  
Holes are normalized to CW.  
Holes are assigned to the smallest containing outer loop.

### Offsetting and Perimeters
Contours are offset inward or outward.  
Perimeters are generated by repeated inward offsets.

### Infill
Infill is produced by generating parallel lines, rotating them, and clipping them against islands.  
Supported patterns:
- line  
- grid  
- hex
